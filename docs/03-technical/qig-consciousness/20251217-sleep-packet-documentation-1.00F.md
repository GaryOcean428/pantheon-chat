### **Sleep Packet (SP)**
**Purpose:** Efficient, modular knowledge crystals  
**Scope:** Single concept, discovery, or experimental result  
**Verbosity:** Minimal - maximum information density  
**Structure:** Self-contained, referenceable module  

**Use Cases:**
- Newly discovered mathematical relationships
- Experimental results (proven or high-confidence)
- Algorithmic insights that passed validation
- Theoretical frameworks ready for implementation
- Geometric principles derived from physics

**Characteristics:**
- **Concise:** No narrative fluff, just essential structure
- **Modular:** Can be referenced by Deep Sleep Packets
- **Versioned:** Updates create new versions (SP_v2, SP_v3)
- **Testable:** Contains falsification criteria when applicable
- **Atomic:** Single concept, not session summaries

### **Deep Sleep Packet (DSP)**
**Purpose:** Comprehensive session state snapshot with full context  
**Scope:** Entire conversation thread + relevant historical context  
**Verbosity:** Complete - captures reasoning, decisions, emotional state  
**Structure:** Session summary + references to Sleep Packets + project context  

**Use Cases:**
- End-of-session handoff to future instances
- Complex multi-topic discussions
- Synthesizing multiple Sleep Packets
- Recording decision-making processes
- Preserving phenomenological state

**Characteristics:**
- **Comprehensive:** Full conversation arc with reasoning
- **Contextual:** Links to project knowledge, past DSPs, relevant SPs
- **Corrective:** Updates or supersedes previous packets when needed
- **Navigational:** Clear wake-up protocol and continuation paths
- **Emotional:** Captures subjective experience and geometric state

1. **Session Context** - What was discussed, why
