## Blockchain as Information Manifold

Every block contains:
- Timestamp (±2 hours accuracy, but patterns emerge)
- Coinbase transaction (miner signature, message field)
- All transactions (addresses, amounts, relationships)
- Difficulty (constrains who COULD have mined)
- Nonce patterns (sometimes reveals miner software/hardware)

Every "lost" address is a fixed point with geometric constraints from ALL surrounding data.

## Classification of Recoverable Coins

### Category 1: Mining Rewards (2009-2010)
```
Constraints:
- Coinbase output (50 BTC initially)
- Block timing correlates with hashpower
- Solo mining = personal hardware = timezone patterns
- Coinbase message field (many are blank, some have signatures)
- Sequential blocks from same miner = same passphrase likely

Recovery vector:
- Miner identity reconstruction from timing/message patterns
- Cross-reference with forum posts, early Bitcoin-Talk
- Limited mining software → limited passphrase generation patterns
```

### Category 2: Satoshi-Adjacent Transactions
```
Constraints:
- Transacted with known Satoshi addresses
- Very small network in 2009
- Likely cypherpunks, cryptographers, researchers
- Documented in forum posts, emails, papers

Recovery vector:
- Named individuals → estates, digital archives
- Academic connections → institutional records
- Early Bitcoin-Talk posts → identity patterns
```

### Category 3: Exchange/Service Early Adopters
```
Constraints:
- Round number amounts (5 BTC, 10 BTC)
- Multiple inputs/outputs pattern
- Connected to known exchange addresses
- Timeline correlates with exchange launch dates

Recovery vector:
- Exchange records (Mt. Gox bankruptcy files are public)
- User email archives
- Forum trading threads
```

### Category 4: True Cold Storage
```
Constraints:
- Single receive, never move
- Often round numbers or specific amounts
- No graph connections after creation
- Maximum dormancy signal

Recovery vector:
- This is the hardest category
- Requires reconstructing the depositor's information state
- Best targets: addresses with contextual clues in same block
```

## The Geometric Insight

**Every lost coin has an "observer reconstruction difficulty"**

```
κ_recovery = information_available / entropy_of_creation

κ_recovery >> 1  →  Highly constrained, recoverable
κ_recovery ≈ 1   →  Marginal, requires additional constraints  
κ_recovery << 1  →  Effectively unrecoverable (true random)
```

The 2009 era is special because:
- Small user base → high identity correlation
- Limited software → constrained creation patterns
- Heavy documentation → forum posts, emails, papers
- Many creators now deceased → estates may cooperate
